## Recording Specifications

### Screen Recording Settings
```
Resolution: 1920x1080 (4K if storage permits)
Frame Rate: 60fps for smooth interface interactions
Bitrate: 50-100 Mbps for crisp detail
Format: MP4 H.264 for compatibility
Audio: 48kHz/24-bit WAV files separate from video
```

### Browser Setup for Recording
```
Browser: Chrome/Firefox with clean profile
Window Size: 1680x1050 (allows for editing margins)
Extensions: Minimal, disable unnecessary UI elements
Zoom Level: 100% for accurate interface representation
Bookmarks: Hidden for clean appearance
```

### Voiceover Recording Standards
```
Microphone: Condenser mic with pop filter
Environment: Quiet room, soft furnishings for acoustic treatment
Format: WAV 48kHz/24-bit for maximum quality
Delivery: Natural pace, clear articulation, consistent energy
Script: Mark pauses, emphasis, and sync points
```

## Post-Production Workflow

### Editing Timeline Structure
```
Track 1: Main screen recording (primary visual)
Track 2: B-roll footage (cutaways, close-ups)
Track 3: Graphics overlay (callouts, annotations)
Track 4: Lower thirds (text identification)
Track 5: Primary audio (voiceover)
Track 6: Background music (subtle, professional)
Track 7: Sound effects (interface sounds, transitions)
```

### Graphics Package Requirements

#### Lower Thirds Template
```
Design: Clean, modern typography
Colors: Talon brand colors (#3B82F6 primary)
Content: Feature names, technical specifications
Animation: Subtle fade-in/out, no distracting motion
Positioning: Bottom left, doesn't obscure interface
```

#### Callout Annotations
```
Style: Subtle highlights, arrows, circles
Color: Bright accent color for visibility
Animation: Gentle pulse or fade-in
Usage: Highlight specific interface elements
Timing: Synchronized with voiceover mentions
```

#### Transition Effects
```
Type: Clean wipes, fades, zoom transitions
Duration: 0.5-1 second maximum
Style: Professional, not flashy
Usage: Scene changes, before/after comparisons
Consistency: Same transition style throughout series
```

### Audio Post-Production

#### Voiceover Processing
```
1. Noise Reduction: Remove background hiss/hum
2. EQ: Enhance vocal clarity, reduce muddy frequencies
3. Compression: Even out volume levels
4. De-essing: Reduce harsh sibilant sounds
5. Normalization: Consistent volume across videos
```

#### Background Music Selection
```
Genre: Corporate/tech ambient, upbeat but not distracting
Volume: -20dB below voiceover, subtle presence
Length: Loop-able tracks for consistent feel
License: Royalty-free or Creative Commons
Style: Modern, professional, matches tech content
```

#### Sound Effects Library
```
Interface Sounds: Subtle click, hover, notification sounds
Transitions: Whoosh, subtle swipe sounds
Success Sounds: Gentle notification, completion chimes
Error Sounds: Warning beeps, alert tones
Volume: Very subtle, enhances but doesn't distract
```

## Quality Control Checklist

### Technical Quality Assurance
- [ ] **Video Quality**: Sharp, no compression artifacts
- [ ] **Audio Quality**: Clear voiceover, no background noise
- [ ] **Color Consistency**: Proper contrast, readable text
- [ ] **Frame Rate**: Smooth 60fps playback
- [ ] **Resolution**: Full HD minimum, 4K preferred

### Content Accuracy Review
- [ ] **Script Adherence**: Voiceover matches approved script
- [ ] **Technical Accuracy**: All demonstrations work as shown
- [ ] **Brand Consistency**: Logo placement, color usage correct
- [ ] **Timing**: Video length within target duration
- [ ] **Information**: All claims verified and up-to-date

### Accessibility Standards
- [ ] **Captions**: Accurate subtitles for hearing impaired
- [ ] **Audio Description**: Visual elements explained in narration
- [ ] **Contrast**: High contrast for screen reader compatibility
- [ ] **Text Size**: All on-screen text large enough to read
- [ ] **Color Coding**: Information not dependent on color alone
